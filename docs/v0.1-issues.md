## Issue #2: Widget State Protocol & Display Palette

**Labels:** `needs-triage`

**Summary:** Define the canonical data structures for Widget state, display primitives, and the connector→core→render message format. This is the "type system" all other issues depend on.

**Description:**

Implement (as Python dataclasses/Pydantic models) the universal Widget state:

```python
# vibe_deck/core/types.py

@dataclass
class DisplayState:
    """What a Widget looks like right now."""
    icon: str           # emoji or key name from icon set
    color: str          # hex background color
    animation: str      # "none" | "pulse" | "crawl" | "blink" | "progress"
    label: str          # short text overlay (max 12 chars)
    badge: str | None   # optional numeric badge

@dataclass  
class WidgetState:
    """A Widget's full state at a point in time."""
    id: str
    type: str           # "agent" | "system" | "command" | "approval"
    display: DisplayState
    meta: dict          # arbitrary adapter-specific metadata

@dataclass
class LayoutFrame:
    """Snapshot of the entire Deck at one moment."""
    deck_type: str         # "Stream Deck XL" etc.
    deck_size: tuple[int,int]  # rows, cols
    widgets: dict[str, WidgetState]  # widget_id → state
    keymap: list[str | None]  # index → widget_id (ordered by physical key position)
```

**Acceptance Criteria:**
- [ ] Pydantic v2 models with JSON schema export
- [ ] `LayoutFrame.from_yaml(path)` + `.to_yaml(path)`
- [ ] `WidgetState` serializes/deserializes round-trip
- [ ] Unit tests for all models

## Issue #4: Core Daemon — Event Loop + Message Bus + Layout Engine

**Labels:** `needs-triage`

**Summary:** The central nervous system. An asyncio-based daemon that starts/stops connectors, routes messages, owns the layout state, and pushes frames to render targets.

**Description:**

```
                    ┌──────────────┐
      Connectors → │ Message Bus  │ → Render Targets
      (inputs)     │  (asyncio Q) │   (outputs)
                    │              │
      Web API ←─── │ Layout Engine│ ──→ Web Editor
                    └──────────────┘
```

Implement:
1. **Message Bus** — `asyncio.Queue` per connector type. Unified internal message format.
2. **Layout Engine** — loads YAML layout, maintains `LayoutFrame` snapshot, recomputes on state change.
3. **Supervisor** — starts/stops all asyncio tasks (connectors, web server, render). Graceful shutdown with `SIGTERM`.
4. **Web API** — FastAPI/aiohttp endpoints: SSE push, WebSocket for real-time, REST for CRUD.

**Acceptance Criteria:**
- [ ] `vibe-deck serve` starts daemon, responds to SIGTERM
- [ ] Layout YAML loaded, parsed, and served to connected simulators
- [ ] SSE push: state change → frame pushed to all connected simulators within 100ms
- [ ] In-memory Widget state updated when connector pushes new state
- [ ] Web API serves simulator static files at `/`
- [ ] `/api/layout` GET returns current `LayoutFrame` as JSON
- [ ] Unit tests for message routing and layout recomputation
