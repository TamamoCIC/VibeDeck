# VibeDeck Adapter Research — Summary

> Generated by `/grill-with-docs` research phase. Four target tools for v0.1 MVP.

## Claude Code

| Aspect | Finding |
|---|---|
| **Official API** | ✅ Hook system — 8 lifecycle events (SessionStart, UserPromptSubmit, PreToolUse, PostToolUse, Stop, PreCompact, SubagentStop, SessionEnd) |
| **Communication** | JSON on stdin to hook scripts. Exit codes 0/1/2 control flow. stdout JSON for modified input and system messages |
| **Status data** | `session_id`, `transcript_path`, `cwd`, `hook_event_name`, `tool_name`, `tool_input` available in stdin |
| **Env vars** | `CLAUDE_CODE_SESSION_ID`, `CLAUDE_TRANSCRIPT_PATH`, `CLAUDE_PROJECT_DIR` exposed to subprocesses |
| **Realtime** | Hook scripts fire inline — VibeDeck adapter can be a hook script that writes status file on each event |
| **Access mode** | File-based: Claude Code config → hook script → writes `~/.vibe-deck/agents/claude-code.json`. Inotify picks it up |

**MVP integration**: Write a Claude Code hooks config (`hooks.json`) that fires a VibeDeck adapter script on SessionStart (status: running), Stop (status: idle/waiting), and UserPromptSubmit (status: waiting_for_user). The adapter emits a JSON status file. VibeDeck watches that file.

**Key events for display primitives:**
| Hook Event | → Widget State |
|---|---|
| SessionStart | Running (🐙 crawling) |
| Stop | Idle (🐙 sleeping) |
| UserPromptSubmit | Waiting for user (🐙 blinking) |
| PreToolUse (error exit 2) | Error (🔴) |

## OpenCode (SST)

| Aspect | Finding |
|---|---|
| **Official API** | ✅ HTTP Server (`opencode serve`) + full OpenAPI spec at `/doc` |
| **Communication** | REST + SSE (Server-Sent Events). 80+ typed events |
| **Status events** | `session.status` (idle/busy/retry), `permission.asked`, `question.asked`, `session.created/updated` |
| **Realtime** | SSE stream at `/event` (per-instance) and `/global/event` (global). Low-latency push |
| **SDK** | `@opencode-ai/sdk` — auto-generated type-safe JS/TS client |
| **Process** | `opencode` TUI process. Can auto-detect via process matching |

**MVP integration**: Connect to `opencode serve` SSE endpoint. Listen for `session.status` events. Translate `busy` → running, `idle` → idle. Also catch `permission.asked` → needs approval Widget.

**Key events for display primitives:**
| SSE Event | → Widget State |
|---|---|
| session.status: busy | Running (🦊 crawling) |
| session.status: idle | Idle (🦊 sleeping) |
| permission.asked | Approval pending (🟡 flash) |
| question.asked | Waiting for user answer (🟡) |

## OpenClaw

| Aspect | Finding |
|---|---|
| **Official API** | ✅ Gateway WebSocket + App SDK |
| **Communication** | Typed WebSocket protocol at `ws://127.0.0.1:18789`. 3 frame types: req/res/event |
| **Status events** | `agent` (start/end/delta), `presence`, `health`, `shutdown`, `tick` |
| **SDK events** | `run.started/completed/failed/cancelled/timed_out`, `tool.call.*`, `approval.requested/resolved` |
| **Realtime** | WS push — built for external monitoring |
| **Auth** | Shared-secret token or password in `connect` params |
| **Process** | Gateway daemon. Detectable via process matching |

**MVP integration**: Connect to Gateway WebSocket. Subscribe to `agent` and `approval.*` events. Translate run lifecycle → Widget animation. Catch `approval.requested` → push to approval zone.

**Key events for display primitives:**
| WS Event | → Widget State |
|---|---|
| agent: start | Running (🦞 crawling) |
| agent: end | Idle (🦞 sleeping) |
| agent: error | Error (🔴) |
| approval.requested | Approval pending (🟡 flash) |
| presence: offline | Inactive (⚫) |

## Telegram

| Aspect | Finding |
|---|---|
| **Official API** | ✅ Telethon (Python MTProto client) + Telegram Bot API |
| **Communication** | Telethon: event-driven async Python. Bot API: webhook or polling |
| **Status data** | New messages per chat, unread counts, online presence of contacts, typing indicators |
| **Realtime** | Telethon `@client.on(events.NewMessage)` handler — persistent connection, not polling |
| **Headless** | ✅ Telethon auth once → `.session` file → run as systemd service, no GUI needed |
| **Access mode** | Adapter script runs a Telethon client as a daemon. Writes status file to `~/.vibe-deck/agents/telegram.json`. VibeDeck watches that file |

**MVP integration**: Small Telethon daemon that tracks unread count per chat + new message notifications. Writes summary to file. VibeDeck watches it.

**Key events for display primitives:**
| Telethon Event | → Widget State |
|---|---|
| NewMessage | New notification (💬 + count badge) |
| Unread count > 0 | Attention (🔵 blink) |
| Unread count = 0 | Idle (💤) |
| Chat-specific message | Show chat name + preview text on key |

## Integration Architecture Summary

```
                    ┌─────────────────────────────────┐
                    │          VibeDeck Hub            │
                    │                                  │
                    │  Process Scanner (pgrep match)   │
                    │  File Watcher (inotify)          │
                    │  WS/SSE Client connectors        │
                    └──┬──────┬─────────┬─────────┬────┘
                       │      │         │         │
              ┌────────┘      │         │         └─────────┐
              ▼               ▼         ▼                   ▼
        ┌──────────┐  ┌──────────┐ ┌──────────┐    ┌──────────┐
        │ClaudeCode│  │ OpenCode │ │ OpenClaw │    │ Telegram │
        │ hook.sh  │  │SSE :PORT │ │WS :18789 │    │ Telethon │
        │ → file   │  │ → stream │ │ → stream │    │ daemon   │
        └──────────┘  └──────────┘ └──────────┘    └──────────┘
             │               │           │               │
             ▼               ▼           ▼               ▼
        ~/.vibe-deck/agents/<name>.json   or real-time WS/SSE connection
```

| Tool | Channel | Integration Complexity |
|---|---|---|
| Claude Code | File watch (hook script writes JSON) | 🟢 Low |
| OpenCode | SSE connect (`opencode serve`) | 🟢 Low |
| OpenClaw | WebSocket connect (Gateway) | 🟢 Low |
| Telegram | Telethon daemon writes file | 🟡 Medium |
